Chapter_05:数组，排序，查找

（1）：数组

数组介绍：

数组可以存放多个同一类型的数据。数组也是一种数据类型，是引用类型。

数组的使用方式（1）：动态初始化

  1. 数组的定义 （在声明的同时创建数组）
    数据类型 数组名[] = new 数据类型[大小]
    例如int a[] = new int[5]意为创建了一个名为a的数组，可以存放五个int类型的）
  2. 数组的引用（引用其中的元素/值）
    数组名[下标]
    例如a[2]意为使用a数组的第三个元素

数组的使用方式（2）：动态初始化

数组的定义（先声明数组，再创建数组）

数组的使用方式（3）：静态初始化

初始化数组：
数据类型 数组名[]={元素值,元素值,元素值,}
数据类型[] 数组名={元素值,元素值,元素值,}
String strs[]=new String[]{"a","b","c"};

细节：

  1. （1）：数组元素的默认值，如int a[]=new int[3]在未对数组元素进行赋值时，
  2. （2）：表示数组大小的数字可以换成变量
  3. （3）：在仅声明数组变量的语句中int a[];这时内存中没有为数组a开辟内存空间，此时的a为null
  4. （4）：使用数组的步骤

    1：声明数组并开辟空间
    2：给数组各个元素赋值
    3：使用数组

  5. 数组属于引用类型，数组型数据是对象
  6. 数组下标必须在指定范围内使用，否则报：下标越界异常
  7. 数组中的元素：满足自动类型转换。即double类型的数组可以存储int类型的元素（自动类型转换），而int类型的数组不可以存储double类型的元素
  8. 数组中的元素可以是任何数据类型，包括基本类型和引用类型

（2）：数组赋值机制

基本数据类型赋值（值传递），这个值就是具体的数据，而且相互不影响。int n1=2; int n2=n1; 即是将2赋给n1，再将n1的值赋给n2(n2的变化不会影响到n1)

数组在默认情况下是引用传递，赋的值是地址。（引用传递）

理解：将array1数组指向的内存地址赋值给了array2，所以此时两个数组变量指向同一个内存空间。此时对array2进行修改，会影响到array1

对 值传递和 引用传递 的深入研究

  1. jvm中的内存分为三大块（1）：栈（2）：堆（3）：方法区
  2. 对于值传递，在栈中int n1=10; int n2=n1;
    表现为n1->[10],n2->[10]将n1指向的10拷贝一份给n2（即n1和n2指向的是不同的内存空间）
  3. 对于引用传递，在栈中int[] arr1={1,2,3};表现为arr1-->0x0011(一个地址)该地址指向堆中的一个相应的空间来存储数组的元素。
    当进行int[] arr2=arr1;时，在栈中，arr1把他的地址拷贝了一份传给arr2arr2-->0x0011
    所以此时，两个数组变量指向同一个内存空间

（6）：排序

排序介绍：

排序是将一群数据，依指定的顺序进行排列的过程

排序的分类

  1. （1）：内部排序：指将需要处理的所有数据都加载到内部存储器中进行排序。包括（交换式排序法，选择式排序法，插入式排序法）
  2. （2）：外部排序：数据量过大，无法全部加载到内存中，需要借助外部存储进行排序。包括（合并排序法，直接合并排序法）

冒泡排序

把大的放到一边：
第一轮把最大的数放在最边缘
第二轮把剩下n-1个数中最大的放在第二个位置
….
(第i轮确定第i大（小）的数在左起（右起）的第i个位置)

冒泡排序优化：

还可以加上一个检索机制，如果在某一轮排序中，没有进行元素顺序的交换，就证明该组数据已经排序完成，则可以直接break。节省时间
