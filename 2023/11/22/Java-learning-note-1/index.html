Chapter01_Java快速入门

01_输出hello world

笔记：

1.public class Hello 表示创建一个public的类(class)Hello

2.Hello{ } 表示一个类(Hello类)的开始和结束
3.public static void main(String[] args) 表示一个main方法(进入程序的入口)
4.main(){ } 表示方法的开始和结束
5.System.out.println("hello world") 表示输出hello world到屏幕
6.;表示语句结束
7.运行时输入java Hello而不是Java Hello.class因为本质上运行的是Hello这个类

02_java运行机制

1..java文件(源文件)--javac(编译)-->.class文件(字节码文件)--java(运行)-->结果

—-(运行的本质):将class文件装载到虚拟机里运行

编译：通过编译器将Java源文件编译成JVM可以识别的字节码文件
每次修改后都要进行保存，并且用javac重新编译

03_Java开发细节

1. Java源文件以 .java 为扩展名。源文件的基本组成部分是类(class)。如上文写到的Hello类

2. Java应用程序的执行入口是main()方法。固定书写格式public static void main(String[] args){...}

3.Java方法由一条条语句构成，每个语句由**;**结束

4. 一个源文件最多只能有一个public类，其他类的个数不限。

运行javac Hello.java得到Hello.class,Cat.class,Dog.class—>编译后，每一个类都对应一个.class文件

也可以将main方法写在非public类中，然后指定运行非public类，这样入口方法就是非public的main方法

javac Hello.java编译后执行java Cat/java Dog分别输出hello cat/hello dogqr

5.如果源文件包含一个public类，则文件名必须按该类名命名

04_Java转义字符

1.\t:一个制表位，实现对齐的功能
2.\n:换行符,不满4个字符位时先补齐然后再空四位
3.\\:一个 \
4.\“:一个 “
5.\‘:一个 ‘
6.\r:一个回车–>光标返回，覆盖上文
System.out.println("123456\r789")
输出结果为789456

05_注释（提高可读性）（先用注释写明每一步的作用，再用代码实现）（plan before programing）

单行注释

“//注释文字”

多行注释

“/*注释文字*/“多行注释里面不允许有多行注释嵌套

文档注释–注释内容可以被JDK提供的工具javadoc所解析，生成一套以网页文件形式体现的该程序的说明文档，一般写在类（Java本质上是在执行类）

  1. 基本格式
  2. 生成对应的文档注释
  3. javadoc标签（暂时用不上到时候自己去查）

Java代码规范（暂时）

类，方法的注释，要以javadoc的方式来写

非javadoc类的注释，往往是给代码的维护者看的。其中要包括

  1. 实现了什么功能
  2. 为什么要这么写

06_补充：

JDK = JRE + Java开发工具

JRE = JVM + 核心类库

path环境变量 ： 当前执行的程序在当前目录下如果不存在,系统会在系统中已有的一个名为path的环境变量指定的目录中查找。
