Chapter04_控制结构

（1）顺序控制：

顺序控制介绍：

程序从上到下逐行的执行，中间没有任何判断和跳转

顺序控制注意事项

Java中定义变量时采用合法的前向引用，即先定义变量，再使用变量

（2）分支控制：

总览：

  1. 单分支
  2. 双分支
  3. 多分支

（1）：单分支

说明（1）：当条件表达式为true时，就会执行{}内的代码，然后执行接下来的代码。如果为false，就不执行{}内的代码，直接执行接下来的代码。
说明（2）：如果{}中只有一条语句，则可以不用{}（大括号将多条语句合并成一个复合语句）

（3）：多分支

基本语法：

说明：

说明（1）：前面的表达式不成立，才轮到对后面的表达式进行判断
说明（2）：一旦有一个表达式成立，执行完代码块中相应的语句后就结束这个分支，进入后续的代码
说明（3）：当所有表达式都不成立时，则执行else的代码块

（3）switch分支结构

基本语法：

说明：

说明（1）：表达式对应一个值
说明（2）：case 常量1：当表达式的值等于常量1，就执行语句块1
说明（3）：break：表示退出switch，如果不加break，就会沿着语句块一路运行下去
说明（4）：如果表达式的值和常量1相等，就执行语句块1，如果不相等，就继续与常量2进行匹配
说明（5）：如果一个case都没有匹配上，则执行default语句块

细节：

  1. 细节（1）：表达式数据类型，应该和case后的常量类型一致，或者表达式的数据类型可以通过自动转换编程常量的数据类型
  2. 细节（2）：switch(表达式)中表达式的返回值必须是：***[byte,short,int,char,String,enum]***
  3. 细节（3）：case中的值必须是常量（或常量表达式），不能是变量
  4. default语句是可选的，当case语句全都不匹配时，执行default
  5. break语句用于执行完一个case分支后跳出switch语句块，如果没有写break，程序会顺序执行到switch结尾

(4)for循环：

基本语法：

细节

  1. for循环执行顺序：（1）：循环变量初始化[只在for循环刚开始时执行一次]
    （2）：判断循环条件
    （3）：执行语句
    （4）：循环变量迭代——–>(2)(3)(4)
  2. 循环条件是返回一个布尔值的表达式
  3. for(;循环判断条件;)中的初始化和变量迭代可以写到其他地方，但是两边的分号不能省略
    在for()内声明的循环变量作用域只在for语句中
  4. 循环初始值可以有多条初始化语句，但要求类型一样，并且中间用逗号隔开（可以声明多个循环变量）。循环变量的迭代也可以有多条变量迭代语句，中间用逗号隔开。
  5. for(;;){}无限循环

(5)while循环

基本语法

细节

  1. 细节（1）：循环条件是返回一个布尔值的表达式
  2. 细节（2）：while循环是先判断再执行语句

（6）do-while循环

基本语法

细节

  1. 细节（1）：和while语句不同do-while语句先执行再判断（判断是否继续执行）也就是说至少执行一次
  2. 细节（2）：最后有一个分号;
  3. 细节（3）：循环条件是返回一个布尔值的表达式
  4. 细节（4）：和while语句相比只是少了对第一次语句执行的判断

(7)跳转控制语句break

基本介绍：

break语句用于终止某个语句块的执行，一般使用在switch或者循环中

细节：

  1. 细节（1）：在for循环中，break语句结束循环时，不会进行循环变量的自增/自减
  2. break语句出现在多层嵌套的语句块中时，可以通过标签指明要终止的是哪一层的语句块
  3. 标签的基本使用
  4. label1是标签，可以任意写
  5. 如果没有指定break，默认退出最近的循环体
  6. break后指定到哪个标签就退出哪个循环
  7. 尽量少用标签（可读性差）

(8)跳转控制语句continue

基本介绍

  1. continue语句用于结束本次循环，继续执行下一次循环
  2. continue语句出现在多层嵌套的循环语句体中时，可以通过标签指明要跳过的是哪一层循环。和前面的break的标签使用规则相同

细节

  1. for循环中如果执行了continue语句，还是会进行变量的自增/自减

(9)跳转控制语句return

介绍

return使用在方法上，表示跳出所在的方法。（如果return写在main方法，退出程序）
