Chapter_07:面向对象编程（中级部分）

编译后在项目文件下产生一个out文件，其中存放源代码编译后的.class文件，源代码中有几个类，就有几个.class文件

（1）：包：

（1）包的三大作用：

  1. （1）：区分相同名字的类（将同名的类放在不同的包中）
  2. （2）：当类很多时，可以很好的管理类（放在不同的包中归类管理）
  3. （3）：控制访问范围

（2）：包的基本语法：

package com.lcxdsg（包com，子包xiaoming）
说明：

  1. package是关键字，表示打包
  2. com.lcxdsg：表示包名

（3）：包的本质分析（原理）

实际上就是创建不同的文件夹（目录）来保存类文件

其中文件夹aaa，bbb分别代表包，且允许不同包中出现相同名字的类

（4）：实例分析


在source文件下开辟文件夹（包）com以及两个子文件夹（子包）xiaoli和xiaoyu，并在两个包中分别定义一个Dog类

当使用不同包中的同名类时，后面的同名类会有具体包的指向（此时不用引用），防止出现混淆。且同名的类只能引用一次

（5）：包的命名

  1. （1）：命名规则：只能包含数字，字母，下划线，小圆点。不能以数字开头，不能是关键字或保留字
  2. （2）：命名规范：一般是小写字母+小圆点
    且一般是com.公司名.项目名.业务模块名

（6）：常用的包

  1. (1):java.lang.*lang包是基本包，默认引入，不需要再引入
  2. (2):java.util.*util包，系统提供的工具包，工具类（Scanner）
  3. (3):java.net.*网络包，网络开发
  4. (4):java.awt.*是做java的界面开发，GUI

（7）：如何引入包（最好需要用到哪个类就引入哪个类）

语法import 包.类具体引入包中的某一个类
import 包.*如import java.util.*引入这个包中的所有类–>引入这个包

（8）：注意事项

  1. (1):package的作用是声明当前类所在的包，需要放在类的最上面，一个类中最多只有一句package
  2. (2):import指令位置放在package的下面，在类定义前面，可以有多句且没有顺序要求

（2）：访问修饰符

基本介绍：Java提供四种访问控制修饰符号，用于控制方法和属性的访问权限（访问范围）

  1. （1）：公开级别：用public修饰，对外公开
  2. （2）：受保护级别：用protected修饰，对子类和同一个包中的类公开（同一个包中的类和不同包中继承的子类可以直接调用）
  3. （3）：默认级别：没有修饰符号，向同一个包的类公开
  4. （4）：私有级别：用private修饰，只有类本身可以访问，不对外公开

注意事项：

  1. （1）：修饰符可以用来修饰类中的属性，方法以及类
  2. （2）：只有默认和public才能修饰类，并遵循上述访问权限的特点
  3. （3）：方法的访问规则和属性一样

特别注意：方法和属性都有四种访问修饰符，且遵守同一个规则。类有public和默认两种访问修饰符，且遵守同一个规则

（3）：面向对象的三大特征：一：封装

封装介绍：

封装就是把抽象出的数据[属性]和对数据的操作[方法]封装在一起，数据被保护在内部，程序的其他部分只有通过被授权的操作[方法]，才能对数据进行操作

理解：

封装的作用在于将功能的具体实现抽象成一个方法，以后要再次实现这个功能时，只需要知道这个方法，不再需要关注方法的实现

封装的实现步骤

  1. （1）：将属性进行私有化[不能直接修改属性]
  2. （2）：提供一个公共的set方法，用于对属性判断并赋值
  3. （3）：提供一个公共的get方法，用于获取属性的值 向get和set方法中分别加入权限判断和数据验证的业务逻辑也属于封装的优点
    理解：将所有属性进行私有化，只能通过特定的方法对其进行操作（限制自由度的同时降低了出错的概率）
    （快速实现get和set方法的快捷键 alt+insert）

封装与构造器：(将构造器与setXxx结合)

直接使用构造器为对象中的属性赋值会绕过setXxx方法—>解决思路，在构造器中调用setXxx方法

（4）：面向对象的三大特征：二：继承

继承的本质：建立访问关系

继承的理解：

将某一个类细分为更加具体的子类时，子类往往都会具有许多主类的属性和方法，但是不同子类之间存在差别。因此就可以通过继承从子类调用主类的某些属性和方法。
避免了方法和属性的多次重写，同时方法的所属范围也条理清晰。

继承的基本介绍：

当多个类存在相同的属性和方法时，可以从这些类中抽象出父类，在父类中定义这些相同的属性和方法，所有的子类不需要重新定义这些属性和方法，只需要通过extends来声明继承父类。在抽象(类)的基础上再次进行抽象(父类)

继承的分析图


子类的子类会同时继承，父类和父类的父类中的属性和方法(public和protected)

继承的基本语法

  1. 子类就会自动拥有父类定义的属性和方法(public和protected)
  2. 父类又叫超类，基类。子类又叫派生类

继承的使用细节：

  1. （一）：子类继承了所有的属性和方法，但是私有属性不能在子类直接访问(public和protected)，要通过父类提供公共的方法去访问(如get方法)
  2. （二）：子类必须调用父类的构造器，完成父类的初始化
  3. （三）：当创建子类对象时，不管使用子类的哪个构造器，默认情况下总会先去调用父类的无参构造器，如果父类没有提供无参构造器（默认或声明），则必须在子类的构造器中用super去指定使用父类的某个构造器完成对父类的初始化工作。
  4. （四）：如果希望指定去调用父类的某个构造器，则显示的调用一下super(参数列表);
  5. （五）：super()在使用时，必须放在构造器第一行（先执行父类的构造器，再执行子类的构造器）（当运行到super()函数的时候才跳转到父类相应的构造器）
  6. （六）：super()只能在构造器中使用
  7. （七）：super()和this()都只能放在构造器的第一行，因此这两个方法不能共存在一个构造器（用了this()的构造器不会有默认super()的执行）
  8. （八）：Java中的所有类都是Object类的子类
  9. （九）：父类构造器的调用不限于直接父类，将一直往上追溯直到Object类(顶级父类)。举例：
  10. （十）：子类最多只能继承一个父类（直接继承），Java中是单继承机制
  11. （十一）不能滥用继承，两个类之间是否满足从属关系

super关键字

基本介绍：super代表父类对象的引用，用于访问父类的属性，方法，构造器

作用/好处：

  1. 调用父类的构造器的好处：分工明确，父类属性由父类初始化，子类的属性由子类初始化
  2. 使用super调用父类的属性与方法的好处：当父类与子类中存在同名属性时可以准确调用，且调用时明确属性的所处类。

基本语法：

super使用细节

  1. 当子类中有和父类中的属性或方法重名时，为了访问父类的成员，必须通过super；如果没有重名，使用super（super不能用于调用本类中的属性和方法），this，以及直接访问效果都一样
  2. super的访问不限于直接父类，如果爷爷类和本类中有同名的成员（属性或方法），也可以使用super去访问爷爷类的成员，如果多个基类（上级类）中都有同名的成员，使用super访问遵循就近原则（先找到谁就访问谁）

方法的重写/覆盖：

基本介绍：

  1. 覆盖：方法覆盖(重写)就是子类有一个方法，和父类的某个方法的名称，返回类型，参数一样，那么我们就说子类的这个方法覆盖了父类的方法。（根据方法的访问原则，会先访问子类中的同名方法，所以就相当于父类的方法被子类中的方法覆盖）（但父类的原方法仍可以通过super调用）
  2. 重写：并且根据子类和父类的使用规则，父类应该包含共有的方法，子类继承父类的方法并在其基础上补充子类特有的方法。如果在子类中定义一个和父类中返回类型，方法名，参数列表完全一样的方法，就相当于把从父类继承得来的方法进行重新定义（重写）

使用细节：

  1. 子类重写的方法的方法名，形参列表要和父类方法相同
  2. 子类方法的返回类型:和父类方法一样，或者是父类返回类型的子类
    比如父类方法的返回类型是Object，子类方法的返回类型是Stringpublic Object getInfo();和public String getInfo();
  3. 子类方法不能缩小父类方法的访问权限
    如果父类是protected则子类只能是public和protected不能为private或default（这样就缩小了父类方法的访问权限）

关于继承的一些理解：（自己补充）

  1. （一）：不能看作是在子类中重新声明了父类的属性和方法，只是在子类需要调用的时候返回父类去调用相应的属性和方法
  2. （二）：关于构造器，子类中没调用super方法则是默认super()。系统根据形参列表匹配父类中相应的构造器。所以当父类中定义了有参构造器时，系统不会默认定义一个无参构造器，此时调用super()则没有相匹配的构造器
  3. （三）：对于方法的继承的思考，方法本质上位于方法区，其在方法区的地址存放在实例对象对应的内存空间中。所以方法的继承和调用一定程度上与属性相似
  4. （四）：关于构造器的继承，每一次的返回父类都是在运行到super()时进行的，且使用了this()的构造器无法使用super()也不能默认
    • 在不同包的子类方法中不能通过父类中protected的构造器new一个父类的实例对象，因为new一个对象不同于父类其他方法的调用(this.父类方法/super.父类方法)
    • 真正意义上的子类可以访问父类的那个被protected修饰的构造器是子类构造器的那个super（如果父类的构造器为private或是default/friendly，那么不同包的子类就无法通过super调用该构造器）
  5. （五）：super()本质上不是super()函数，而是super关键字代替了父类组成的构造器语句,父类名();–>super();—-同理this
  6. （六）：如果函数名和形参列表有一个不相同就不会认为是重写，因此可以达到一个父子类之间方法的重载（严格意义上是不同的方法，只是这么去理解）

多态

问题：当对于不同的事物执行类似/相同的算法时，需要写很多功能相同但只是换了几个参数的代码（不利于管理和维护）—->引出多态，提高代码复用性

多态的基本介绍:

方法或对象具有多种形态。多态是建立在封装和继承基础之上的

多态的具体体现：（一）：方法的多态

重载和重写体现多态

多态的具体体现（二）：对象的多态:

重要的几句话：

  1. 一个对象的编译类型和运行类型可以不一致，父类的引用可以指向子类的对象
  2. 编译类型在定义对象时，就确定了，不能改变(对象名的类型)
  3. 运行类型是可以变化的—>体现多态
  4. 编译类型看定义时”=”号的左边，运行类型看”=”号的右边

编译类型（编译）和运行类型（执行）

  1. 引用变量在编译阶段只能调用其编译时类型所具有的方法，即编写代码时，只能调用编译类型中具有的方法。程序在编译时，会在编译类型中检查是否具有所调用的方法，如果编写代码时，使用引用变量调用子类中的特有方法，或者调用重载了父类中的方法，而父类中找不到该方法，则会报编译错误
  2. 但运行时则执行它运行时类型所具有的方法，如果子类重写了该方法，运行时实际调用的是运行时类型的该方法。

应用：给动物喂食

向animal传入其不同子类，向food传入其不同子类。（形参列表中定义的是编译类型【对象名/对象引用】，传入的是运行类型【对象】）
极大提高了代码的复用性

多态注意事项和细节

1.多态的前提：两个对象(类)存在继承关系，或是方法存在重载

2. 多态的向上转型：

  1. 本质：父类的引用指向了子类的对象
  2. 语法：父类类型 引用名=new 子类类型(形参列表);（不要求直接父类，只要是父类就行）
  3. 特点：

    （一）：编译类型看左边，运行类型看右边
    （二）：可以调用父类中的所有成员，需遵守访问权限（子类可以从父类中继承）
    （三）：不能调用子类中特有成员（程序在编译时，会在编译类型中查找是否有这种方法，没有则报错）
    （四）：最终运行效果看子类的具体实现（子类是否重写了方法）
    （五）：方法的查找顺序：从子类开始往上找

3. 多态的向下转型

  1. 语法：子类类型 引用名=(子类类型) 父类引用(编译类型为父类的对象引用);(类型的强转)
  2. 只能强转父类的引用，不能强转父类的对象
  3. 要求父类的引用必须指向的是当前目标类型的对象的父类/本类（即转换后对象引用的类型要大于等于其指向对象的类型）（转换完后的类型仍要为其指向对象的类型的父类/同类）
  4. 向下转型后可以调用子类类型中所有的成员（此时编译类型和运行类型相同）

4. 小总结

  1. 和基本数据类型的自动转换与强制转换存在相似之处
  2. 转型后和转型前都指向同一个对象，所以转型前后的对象引用相等

属性重写问题

属性没有重写之说，属性的值看编译类型

属性的访问是基于引用变量的编译类型的，这称为静态绑定。编译时，编译器使用引用变量的类型来确定属性的访问。即使在运行时，引用变量指向了子类对象，也只有编译类型所包含的属性可见。这是因为属性不涉及动态绑定，编译器在编译时已经确定了访问哪个类的属性

以编译类型的对象为起点，向上查找

Java的动态绑定机制

介绍：

  1. 当调用对象方法的时候，该方法会和该对象的内存地址/运行类型绑定（方法的调用都是从运行类型开始的）
  2. 当调用对象属性时，没有动态绑定机制，调用对象引用的编译类型中对应的属性
  3. 在类内部调用属性时，哪里声明，调用哪里（没有特殊声明情况下，如super）（没有就向父类找）

多态数组

基本介绍：数组的定义类型为父类类型，里面保存的实际元素类型为子类类型/同类

（相当于double数组可以存储int和double，包含一个自动类型转换）

细节：

  1. 多态数组中元素的编译类型都为数组的定义类型
  2. 多态数组中调用子类特有的方法（向下转型）

多态参数

基本介绍：

方法定义的形参类型为父类类型，实参类型允许为子类类型/本类

细节：

  1. 传参的过程中包含一个向上转型
  2. 传进去的参数的编译类型为形参列表中定义的类型，运行类型为实参指向的对象的类型

Object类（顶级父类）

equals方法

==：是一个比较运算符

  1. ==：既可以判断基本类型，又可以判断引用类型
  2. ==：如果判断基本类型，则判断值是否相等（10==10.0）
  3. ==：如果判断引用类型，判断的是地址是否相等，即判定是不是同一个对象

equals方法：是Object类中的方法

  1. 只能判断引用类型
  2. 默认判断的是地址是否相等，子类中往往重写该方法，用于判断内容是否相等

hashCode方法

粗略了解

  1. 提高具有哈希结构的容器的效率
  2. 两个引用，如果指向的是同一个对象，则哈希值肯定是一样的
  3. 两个引用，指向的是不同对象，则哈希值是不一样的（不肯定）
  4. 哈希值主要根据地址得来的，但是不能完全将哈希值等价于地址（由Object类定义的hashCode方法会针对不同的对象返回不同的整数）
  5. 后面在集合中，如果需要的话，会重写hashcode

toString方法

基本介绍：（返回该对象的字符串表示）

默认返回：全类名+@+哈希值的十六进制
子类往往重写toString方法，用于返回对象的属性信息

重写toString方法，一般是把对象的属性值输出

当直接输出一个对象时，toString方法会被默认调用

finalize方法（finalize=最后敲定）

基本介绍：

当垃圾回收器确定不存在对该对象的更多引用时，由对象的垃圾回收器调用此方法

作用

通过 finalize 方法让对象在被销毁前留下一些“遗言”来完成某些业务逻辑

细节：

  1. （一）：当对象被回收时，系统自动调用该对象的finalize方法，子类可以重写该方法，做一些释放资源的操作
  2. （二）：什么时候被回收：当某个对象没有任何引用时，JVM就认为这个对象是一个垃圾对象，就会使用垃圾回收机制来销毁该对象，在销毁对象前，会先调用finalize方法
  3. （三）：垃圾回收机制的调用，是由系统来决定（不是一成为垃圾就马上回收的，系统有自己的算法），也可以通过System.gc()主动调用垃圾回收器（但是垃圾回收器不一定理你）
  4. （四）：如果程序员不重写finalize，那么就会调用Object类的finalize方法，即默认处理

断点调试：

断点调试快捷键

  1. F7:跳入方法内/step into（强制跳入方法内shift+alt+F7）
  2. F8:逐行执行代码/step over
  3. shift+F8:跳出方法/step out
  4. F9:执行到下一个断点/resume

细节补充：

  1. 在断点调试过程中，是运行状态，是以对象的运行类型来执行的
  2. 将光标放在某个变量上，可以看到最新的数据
  3. 追源码，调用方法时，通过F7/step into 进入方法的定义处
  4. 可以在debug的过程中，动态的下断点
  5. 执行到下一个断点时，会根据逻辑判断运行到哪一个断点（比如有一个断点因为不满足条件执行不到了，就会运行到下一个断点）—->可以通过这个特性观察代码能否运行到该断点处

一些理解：

  1. 编译类型，栈中对象名的类型。运行类型，实际上对象名指向的对象的类型
  2. 因为子类其实是一种特殊的父类，因此Java允许把一个子类对象直接赋值给一个父类引用变量，无需任何类型转换
  3. 编写Java代码时，引用变量只能调用声明该变量所用类（编译类型）里包含的方法。与方法不同的是，对象的属性则不具备多态性。通过引用变量来访问其包含的实例属性时，系统总是试图访问它编译时类所定义的属性，而不是它运行时所定义的属性。
    ——要访问子类中特有的方法和属性，在编写代码时，则必须进行类型转换。
  4. 在子类new一个对象的时候，父类的信息在其之前（同时/指顺序）就已加载到方法区，且在子类内部有一块独立的空间存储属性和方法
  6. 在类的内部调用属性时，在不使用super关键字的情况下，该属性在哪调用，就使用在该类中声明的属性（没有就向父类找）
  7. 对象(对象引用)的编译类型和运行类型可以不一致，编译类型在定义时就确定，不能变化。对象的运行类型是可以变化的，可以通过getclass()来查看运行类型GrandFather g=new Father;---g=new Son();（g的编译类型GrandFather无法改变）
  8. 编译类型和运行类型都是针对对象引用（引用变量）的

待解决的问题：

  1. （一）：在子类实例的生命周期里，是否会为子类和父类同时开辟一块内存空间：解答，在子类对象的内存空间中会为父类开辟一块独立的内存空间加载信息
  2. （二）：子类的内存空间中是否存储着从父类继承的属性：存储在父类独立的内存空间中
  3. （三）：一些猜想：在创建子类实例的时候，会先创建一个父类的实例对象，再创建一个子类的实例对象，子类实例的内存空间中并不存储继承的属性，而是从主类对象中调用。：某种意义上说是的，不过父类的“实例对象”在子类的内存空间中
  4. （四）：方法的内存表现，方法加载到方法区，调用的时候转到方法定义处，运行结束后返回方法调用处
  5. （五）：父类和子类构造器什么时候调用？调用的顺序和步骤，如创建对象，显式初始化，构造器初始化…：遵循对象的创建机制，从顶级父类到子类依次在子类的内存中开辟独立的内存空间（创建对象）
