Chapter06_面向对象编程（基础部分）

（1）：类与对象关系:


理解：
（1）：类–>种类，（比如人类）。实例对象–>一个个属于该类的具体的个体（比如我们每一个人）
（2）：类是数据类型，对象就是一个具体的实例
（3）：把一群具有相同性状特征的个体聚集在一起就可以归为一个类
（4）：类/对象，都有属性和行为（方法）两种特性
### 注意：；
从类到对象，有几种说法：
（1）：创建一个对象
（2）：实例化一个对象
（3）：把类实例化
## （2）：对象在内存中存在形式

### （1）类是引用数据类型—>对象属于引用数据类型
### （2）所以在类的实例对象在栈中存储一个地址，该地址指向堆中的一块内存(new开辟的内存空间)【每一个属性都有对应的内存空间】
### （3）在堆中的内存空间里，因为字符串属于引用数据类型，所以堆中的字符串对应的内存空间仍然存储一个地址，该地址指向方法区中的常量池（字符串在常量池中存储）
### （4）对于基本数据类型，则直接存储在堆中为该实例对象开辟的内存空间中

### （5）在执行new操作时，会在方法区加载该类的信息（属性信息和方法信息）（只加载一次）
### （6）堆中的内存空间才是真正的变量空间（真正的对象），对象名只是存储了指向该空间的地址，可以称对象名为“对象引用”
### （7）Person p1 = new Person();其中，new Person()在堆中创建了一个对象，而p1作为对象名/对象引用只是存储了指向这个对象的地址

## （4）：方法
### （1）：创建方法

1. public：表示方法是公开的
2. void：表示方法没有返回值
3. speak：方法名
4. () :形参列表
5. {}：方法体，写要执行的代码
### （2）：调用方法
先创建对象，再调用方法：对象名.方法名

### （3）：方法的调用机制原理

1. 首先执行main()方法：在栈中开辟一个main栈。
2. Person p1 = new Person();在堆中开辟一块内存空间存储对象，并把地址赋值给p1，此时变量p1位于main栈中。同时常量池加载Person类的信息（属性，方法）
3. 当执行到 int res = p1.getSum(10,20);时（先计算右值再赋给左值）意为在执行p1.getSum(10,20)时，会在栈中开辟一块新的，独立的空间（属于getSum方法的栈）。
4. 并且跳转到类中相应的方法定义处
5. 根据位置传参，将实参传入形参（在getSum新栈中，形参被赋予了传入实参的值）
6. return语句返回值到调用处
7. 方法执行完毕后，getSum栈被销毁

小结：
1. （1）：当程序执行到方法时，就会开辟一个新的，独立的空间（栈空间）
2. （2）：当方法执行完毕，或者执行到return语句时，就会返回到调用方法的地方
3. （3）：返回后，开辟的栈随之消失（main方法结束后，开辟的main栈也随之消失，整个程序退出）
4. （4）：返回后，继续执行后面的代码

### （4）：方法的使用细节：
1. （1）：访问修饰符：控制方法使用的范围
[public,private,protected,默认]
2. （2）：方法里不能定义方法

返回值

  1. 一个方法最多有一个返回值,如果要返回多个值—>返回数组（还可以返回一个类的对象，其中的属性可以存储多种类型的数据，并且还有属性名进行标识）
  2. 返回类型可以为任意类型（基本类型，引用类型）
  3. 如果方法要求返回数据类型，则方法体中最后的执行语句必须为return 值;（返回的可以是/常量/变量/表达式）而且要求返回值类型必须和return的值类型一致或兼容。如返回值类型为double，则可以return int
  4. 如果方法是void，则方法体中可以没有return语句，或者只写return ;

形参列表

  1. 一个方法可以有0个参数，也可以有多个参数，中间用逗号隔开
  2. 参数类型可以为任意类型，包含基本类型或引用类型
  3. 调用带参数的方法时，一定对应着参数列表传入相同类型或兼容类型的参数（传参注意参数类型匹配）
  4. 方法定义时的参数称为形式参数，简称形参；方法调用时传入的参数称为实际参数，简称实参。实参和形参的类型要一致或兼容，个数，顺序必须一致。（位置传参）

方法调用说明

  1. （1）：同一个类中的方法调用：直接调用即可
  2. （2）：跨类中的方法调用，在A类调用B类中的方法：需要通过对象名调用
  3. 特别说明：跨类的方法调用和方法的访问修饰符相关

（5）：方法传参机制：

  1. （1）：基本数据类型的传参机制
    main栈和方法栈相互独立，且基本数据类型的传参方式为值传递，相当于把实参的值复制一份给形参，所以形参的变化不会影响到实参
    总结：对于基本数据类型，传递的是值(值拷贝)，形参的任何改变不影响实参
  2. （2）：引用数据类型的传参机制
    虽然main栈和方法栈相互独立，但是引用数据类型的传参方式为引用传递，相当于把实参的地址复制一份给形参，所以位于不同栈中的实参和形参实际上指向“堆”中的同一个对象/内存空间，所以形参的变化会影响到实参（影响到堆中的数据）
    总结：引用类型传递的是地址（传递的也是值，但是值是地址），可以通过形参影响实参

（5）：方法递归调用：

（1）：递归的基本介绍：递归就是方法自己调用自己，每次调用时传入不同的变量。

（2）：递归调用机制：

关键点：每调用一次方法，就开辟一个独立的新栈，程序跳转到方法处继续执行（同一种方法也是如此）
并且方法执行结束后，销毁该栈，返回调用处

递归总是从最后开辟的栈开始返回的

（3）：递归重要规则：

  1. （1）：执行一个方法时，就创建一个新的受保护的独立空间（栈空间）
  2. （2）：方法的局部变量是独立的，不会相互影响
  3. （3）：如果方法中使用的是引用类型变量，就会共享该引用类型的数据
  4. （4）：递归必须向退出递归的条件逼近，否则就是无限递归，出现StackOverflowError
  5. （5）：当一个方法执行完毕，或者遇到return，就会返回，将返回值返回到调用处（相应的栈也销毁了）

（4）：递归的重要思想：找到f(n)与f(n-x)的关系(找到变化规律)

每一次动作的执行都要经历相同的步骤，只要总结出规律，并且找到其中一次的运行过程就可以写出递归算法

结束递归的条件很重要

重载的好处

  1. （1）：减轻了起名的麻烦
  2. （2）：减轻了记名的麻烦
    同一个方法名可以对不同类型的形参列表进行操作

重载的使用细节

  1. （1）：方法名：必须相同
  2. （2）：形参列表：必须不同
  3. （3）：返回类型：无要求

（2）：数组赋值机制

基本数据类型赋值（值传递），这个值就是具体的数据，而且相互不影响。int n1=2; int n2=n1; 即是将2赋给n1，再将n1的值赋给n2(n2的变化不会影响到n1)

数组在默认情况下是引用传递，赋的值是地址。（引用传递）

理解：将array1数组指向的内存地址赋值给了array2，所以此时两个数组变量指向同一个内存空间。此时对array2进行修改，会影响到array1

对 值传递和 引用传递 的深入研究

  1. jvm中的内存分为三大块（1）：栈（2）：堆（3）：方法区
  2. 对于值传递，在栈中int n1=10; int n2=n1;
    表现为n1->[10],n2->[10]将n1指向的10拷贝一份给n2（即n1和n2指向的是不同的内存空间）
  3. 对于引用传递，在栈中int[] arr1={1,2,3};表现为arr1-->0x0011(一个地址)该地址指向堆中的一个相应的空间来存储数组的元素。
    当进行int[] arr2=arr1;时，在栈中，arr1把他的地址拷贝了一份传给arr2arr2-->0x0011
    所以此时，两个数组变量指向同一个内存空间

（6）：排序

排序介绍：

排序是将一群数据，依指定的顺序进行排列的过程

排序的分类

  1. （1）：内部排序：指将需要处理的所有数据都加载到内部存储器中进行排序。包括（交换式排序法，选择式排序法，插入式排序法）
  2. （2）：外部排序：数据量过大，无法全部加载到内存中，需要借助外部存储进行排序。包括（合并排序法，直接合并排序法）

冒泡排序

把大的放到一边：
第一轮把最大的数放在最边缘
第二轮把剩下n-1个数中最大的放在第二个位置
….
(第i轮确定第i大（小）的数在左起（右起）的第i个位置)

冒泡排序优化：

还可以加上一个检索机制，如果在某一轮排序中，没有进行元素顺序的交换，就证明该组数据已经排序完成，则可以直接break。节省时间

进制总览：

  1. 二进制：0，1.满2进1，以0b或0B开头(binary)
  2. 十进制：0-9.满10进1
  3. 八进制：0-7.满8进1，以数字0开头

进制转换(十进制转其他进制)：

原理
(以34为例)：
17(被2除1次) 0(被2除0次)//
8(被2除2次) 1(被2除1次)//
4(被2除3次) 0(被2除2次)//
2(被2除4次) 0(被2除3次)//
1(被2除5次) 0(被2除4次)//
0(被2除6次) 1(被2除5次).
结果为：100010—->1 * 2^5 + 1 * 2^1

  1. 十进制转换成二进制：将该数不断除2，直到商为0，然后将每步得到的余数倒过来
  2. 十进制转换成八进制：将该数不断除8，直到商为0，然后将每步得到的余数倒过来
  3. 十进制转换成十六进制：将该数不断除16，直到商为0，然后将每步得到的余数倒过来

进制转换(二进制转其他进制)

  1. 二进制转八进制：从低位开始，将二进制每三位结合成一组，转成对应的八进制数(2^3=8)
  2. 二进制转十六进制：从低位开始，将二进制每四位结合成一组，转成对应的十六进制数(2^4=16)

进制转换(其他进制转成二进制)

  1. 八进制转换成二进制：从低位开始，将八进制数的每一位，转成对应的一个3位的二进制数
  2. 十六进制转换成二进制：从低位开始，将十六进制数的每一位，转成对应的一个4位的二进制数

(8).位运算符(原码，反码，补码)

  1. 二进制的最高位是符号位:0表示正数，1表示负数
  2. 正数的原码，反码，补码都一样
  3. 负数的反码=它的原码符号位不变，其他位取反(0–>1,1–>0)
  4. 负数的补码=它的反码+1//同理//负数的反码=它的补码-1
  5. 0的反码，补码都是0
  6. java没有无符号数，Java中的数都是有符号的
  7. 在计算机运算的时候，都是以补码的方式来运算的（无论什么运算都是）（且补码运算得到的结果也为补码，要转回原码才能得出准确答案）
  8. 当我们看运算结果的时候，要看他的原码

04_基本数据类型的转换

自动类型转换

  1. 当Java程序在进行赋值或者运算时，精度小的类型自动转换为精度大的数据类型，这个就是自动类型转换
  2. 数据类型排序,按精度(容量)排序
    char-->int-->long-->float-->double
    byte-->short-->int-->long-->float-->double

自动类型转换注意和细节

  1. 有多种数据类型的混合运算时，系统首先自动将所有数据转换成容量最大的那种数据类型，然后再进行计算
  2. 当我们把精度(容量)大的数据类型赋值给精度(容量)小的数据类型时，就会报错。反之将精度(容量)小的数据类型赋值给精度(容量)大的变量时，就会进行自动类型转换。
  3. byte，short和char之间，不会相互自动转换
  4. 将整型常量赋值给byte或short类型时，系统会先判断该类型能否存储该常量。如果是将变量赋值给该变量，就会直接判断类型
  5. byte,short,char类型三者可以计算，在计算时首先转换为int类型，结果为int类型
  6. boolean不参与转换
  7. 自动提升原则：表达式结果的类型自动提升为，操作数中最大的类型

强制类型转换

自动类型转换的逆过程，将容量大的数据类型转换为容量小的数据类型。使用时要加上强制转换符()，但可能造成精度降低或溢出，格外要注意

强制类型转换细节说明

  1. 当进行数据从大–>小，就要用到强制类型转换
  2. 强制类型转换符号”()”只针对于最近的操作数有效，往往会使用小括号提升优先级
  3. char类型可以保存int的常量值，但不能保存int的变量值，需要进行强制类型转换
  4. byte和short，char类型在进行运算时，当作int类型处理

基本数据类型和String类型的转换

  1. 基本数据类型转String类型:将基本类型的值+””
    “+”字符串加其他 = 字符串 Char型可以变换成String型
  2. String类型转基本数据类型:通过基本类型的包装类,调用parseXX方法(parse=解析) 把字符串String转成字符char—>得到字符串的第几个字符
  3. 注意事项:
    （1）：在将String类型转成基本数据类型时，要确保String类型能够转成有效的数据。例如:
    int n1 = Integer.parseInt("hello");—报错
    （2）：如果格式不正确，就会抛出异常，程序就会终止（没能转成有效的数据）

6.;表示语句结束

02_java运行机制

1..java文件(源文件)--javac(编译)-->.class文件(字节码文件)--java(运行)-->结果

—-(运行的本质):将class文件装载到虚拟机里运行

编译：通过编译器将Java源文件编译成JVM可以识别的字节码文件
每次修改后都要进行保存，并且用javac重新编译

03_Java开发细节

1. Java源文件以 .java 为扩展名。源文件的基本组成部分是类(class)。如上文写到的Hello类

2. Java应用程序的执行入口是main()方法。固定书写格式public static void main(String[] args){...}

3.Java方法由一条条语句构成，每个语句由**;**结束

4. 一个源文件最多只能有一个public类，其他类的个数不限。

运行javac Hello.java得到Hello.class,Cat.class,Dog.class—>编译后，每一个类都对应一个.class文件

也可以将main方法写在非public类中，然后指定运行非public类，这样入口方法就是非public的main方法

javac Hello.java编译后执行java Cat/java Dog分别输出hello cat/hello dogqr

5.如果源文件包含一个public类，则文件名必须按该类名命名

4.\“:一个 “
